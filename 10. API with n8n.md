## Part 1: Native Integration vs. HTTP Request Node

n8n has 500+ **native integrations** (Gmail, Sheets, Drive, Tavily, etc.) — these are pre-built wrappers around each service's API. Someone already read the docs and built you a friendly form to fill in instead of writing raw API calls.

### But native integrations are limited in two ways

| Limitation | Example |
|---|---|
| Not every app has one | Your own custom app/backend definitely won't |
| Not every API endpoint is wrapped | Tavily's API has 5 endpoints (search, extract, crawl, map, usage) — but n8n's native Tavily node only supports 3 of them |

### The fix: HTTP Request node
This is the **universal adapter** — it can hit any API on Earth, wrapped or not.

**Key trick: Import cURL**
1. Copy the curl command straight from any API's documentation
2. Paste into the HTTP Request node
3. Click **"Import cURL"**
4. n8n auto-fills the method, URL, headers, and body for you

This is how you can connect to an endpoint with no native n8n support, or even a completely custom API you built yourself.

### Making it dynamic
Just like the drag-and-drop trick from earlier modules, you can turn any hardcoded value in the HTTP node into an expression — e.g., pulling a user's chat message into the JSON body as the search query — so the HTTP Request node behaves like a real, dynamic integration.

> **One-line takeaway**: Native integration = convenience wrapper. HTTP Request node = full control, works with literally anything.

## Part 2: n8n Workflow + APIs (hands-on with n8n's own tutorial)

n8n has a built-in interactive API tutorial workflow — webhooks acting as a pretend restaurant "kitchen." This reinforces the restaurant analogy hands-on.

**Setup step**: Copy your own webhook's production URL and paste it into a "configuration" node — this becomes your base URL, similar to how a country code (+91, +1) gets prefixed to a phone number.

### Lesson-by-lesson breakdown

| Lesson | Concept | What happens |
|---|---|---|
| 1 | Basic GET request | Hit a webhook → get back a menu (plain JSON response) |
| 2 | Query parameters | `?extra_cheese=true` — note: query param values are always **strings**, even `true`/`false`. This is why the If node needs "loose comparison" to correctly treat the string `"true"` as boolean `true` |
| 3 | POST + Body | Sending `{"comment": "amazing", "rating": 5}` — you can't easily do this from a browser URL bar, which is why body exists as a separate channel |
| 4 | Headers + Auth | `x-api-key: <secret>` sent in headers → webhook checks it → wrong key = `401 Unauthorized` |
| 5 | Timeout / Error handling | The HTTP Request node has two output paths: success and error. If the "kitchen" takes 3 seconds but you only wait 2 (`timeout=2000ms`), you get routed down the **error path** instead of hanging forever |

**Real-world parallel**: if a restaurant takes 2 hours to bring your food, you'd cancel — that's exactly what a timeout does programmatically.

n8n's tutorial also hits a totally external API (the Cat Facts API) with a plain HTTP node — no auth needed — to show that everything you learn generalizes to any API, not just the tutorial's pretend one.

## Part 3: Auth in n8n (hands-on deep dive)

This is the practical version of the auth theory — actually configuring each method inside the HTTP Request node.

| Method | How it's set up in n8n | Analogy |
|---|---|---|
| **Predefined Credential** | Pick the service (Gmail, Tavily, GitHub) → n8n handles OAuth automatically | Preloaded gym pass |
| **Basic Auth** | Username + password saved as credential | Showing ID + PIN at a front desk |
| **Digest Auth** | Same idea as Basic, but hashed | OTP-style handshake |
| **Header Auth / Bearer Auth** | You manually create the header key (`Authorization`) and value (`Bearer <token>`) | Flashing a digital keycard |
| **OAuth1 / OAuth2** | Full credential flow — Client ID, Secret, Auth URL, Token URL | Bank-style permission tokens |
| **Query Auth** | Key/value gets appended to the URL as a query param | Yelling your password across the room — **avoid this when possible** |
| **Custom Auth** | Manually combine headers/body/query however a weird API demands | Passing a note in secret code |

### The critical practical lesson

> **There is no universal auth format.** Each API's docs (or its cURL example) tell you exactly how it expects to be authenticated — as a header, as `Bearer <token>`, as a raw key, or as a query param. Guessing wrong wastes hours. Always check the documentation's example request first.

### Bearer Auth vs Header Auth (commonly confused)

- **Bearer Auth** = n8n's dedicated, specialized option. You just paste your token (e.g. `abc123`), and n8n writes the full `Authorization: Bearer abc123` header for you.
- **Header Auth** = the generic, manual version. You'd have to type the key yourself (`Authorization`) AND the value yourself, including typing the literal word `Bearer` — one extra step, more room for a typo (miss the space after "Bearer" and the whole call fails).
- **Query Auth** would be wrong for a `Bearer` token entirely — that's only for when the docs show the key stuck onto the URL itself (`?api_key=abc123`), not inside a header.

Functionally, Bearer Auth and "Header Auth done correctly" produce the identical request on the wire — Bearer Auth just removes the chance of you messing up the formatting.

### Closing lesson worth remembering

> "AI can only help you when you know things." Understanding auth deeply means you can ask an AI tool a much sharper debugging question instead of guessing blindly.

## Key Takeaway

- **Native integrations** = convenience wrappers (limited coverage). **HTTP Request node** = universal, connects to anything.
- **Import cURL** is the fastest way to connect n8n to any API you find in the wild
- Query param values are always **strings** — even `true`/`false` — watch out for this in If/condition nodes
- Timeouts route failed/slow requests down an **error path** instead of hanging forever
- **Always check an API's docs first** for exactly how it wants to be authenticated — there's no universal auth format
- **Bearer Auth** = safer, auto-formatted version of Header Auth for token-based APIs
