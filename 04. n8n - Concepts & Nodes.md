## Quick Recap

This module teaches the **"grammar" of automation** — 6 core terms that apply to n8n (and honestly, any automation tool). Then it explains how n8n organizes its 400+ building blocks ("nodes") into clear categories.

---

## Part 1: Core Concepts & Terminology

These 6 terms are the foundation. Once you know these, you can talk about ANY workflow, in ANY tool (n8n, Zapier, Make — doesn't matter).

### 1. Workflow — "One complete automation journey"
Think of it as a **recipe** — start to finish, all the steps together.
- **Personal example**: Wake up → brush teeth → make coffee → check calendar
- **Work example**: Form submitted → enrich data → send welcome email → add to CRM

### 2. Node — "One single step" (the building block)
Like **one worker on a factory assembly line** — each node does ONE specific job.
- Example: A "Gmail Send" node's *only* job is sending an email. Nothing else.
- **Naming tip**: always name nodes as **Verb + Object** → "Send Email", "Fetch Orders" (makes big workflows much easier to read later)

### 3. Trigger — "What starts the workflow" (the doorbell)
This is always the **first node** — the thing that kicks everything off.

| Trigger type | Think of it like... |
|---|---|
| Manual Trigger | Pressing "play" yourself, for testing |
| Schedule Trigger | An alarm clock — runs at fixed times |
| Webhook Trigger | A doorbell — some outside app "rings" it |
| App Trigger (Gmail, Telegram, etc.) | Something specific happens in that app |

### 4. Action — "What happens after the trigger fires"
If trigger = alarm ringing, action = you actually getting out of bed and making coffee.
- Example: New form submitted (trigger) → send Slack alert (action)

### 5. Data Flow — "The fuel moving between nodes"
Data moves from node to node like a **parcel on a conveyor belt** or a **baton in a relay race**. Each node:
1. Receives data from the node before it
2. Does something with it
3. Passes the (possibly changed) data to the next node

Example: Someone types "Devasish" in a form → that name flows to the next node → gets used to personalize an email → "Hi Devasish!"

### 6. JSON — "The blueprint/format the data travels in"
JSON (JavaScript Object Notation) is a simple **key-value** way of writing data:
```json
{
  "name": "Priya",
  "email": "priya@acme.com"
}
```
This is exactly how n8n workflows themselves get saved and shared — every workflow file is a `.json` file.

### Summary Table

| Concept | Think of it like... | Work Example | Personal Example |
|---|---|---|---|
| Workflow | Recipe / Project plan | Form → enrich → email | Wake up → coffee |
| Node | Factory worker | Gmail Send node | Calendar Create node |
| Trigger | Doorbell / Alarm | Webhook: new form | Telegram: friend texts |
| Action | Cooking step | Send Slack alert | Reply with your schedule |
| Data Flow | Relay baton | Customer ID → fetch details | "7pm movie" → calendar event |
| JSON | Blueprint | Export workflow | Import ready-made workflow |

## Part 2: Nodes Explained (categories)

n8n groups its 400+ nodes into clear categories. When you press `Tab` or click `+`, you'll see these buckets:

| Category | What it does | Examples |
|---|---|---|
| **Trigger** | Starts the workflow | Manual, Schedule, Webhook, Form, Telegram, Gmail |
| **Action in an App** | Does something inside a real app | Gmail, Notion, Google Sheets, AWS, GitHub, Jira |
| **Data Transformation** | Reshapes/filters/cleans your data | Filter, Merge, Aggregate, Remove Duplicates, Split Out |
| **Core** | Generic/technical building blocks | Code (write your own JS/Python), HTTP Request, Webhook |
| **AI** | AI-specific blocks | AI Agent, Basic LLM Chain, Sentiment Analysis, Text Classifier |
| **Human in the Loop** | Pauses for a real person's approval | Send to Slack and *wait* for someone to approve |
| **Flow** | Controls logic/branching | If, Switch (like an if/else or case statement in code) |

### Important details to remember

1. **The "Code" node** — lets you drop into real JavaScript (or Python, still in beta) when no-code isn't enough. It even has an "Ask AI" option where you describe what you want in plain English and it writes the code for you.
2. **The "HTTP Request" node** — this is huge. It means n8n can connect to *literally any* API, including your own custom apps. This is the escape hatch when there's no dedicated node for something.
3. Workflows flow **left to right**, and should be read top-to-bottom, logically — like reading a sentence.
4. **Importing/exporting**: Download a workflow as `.json` → use "Import from file" in a new n8n canvas to load it. This is how workflow files get shared between people.

## Key Takeaway

- **6 core terms** (Workflow, Node, Trigger, Action, Data Flow, JSON) are the "grammar" of automation — learn these once, use them everywhere
- **Trigger vs Action**: Trigger = what starts it, Action = what happens after
- n8n organizes nodes into 7 categories — Trigger, App Action, Data Transformation, Core, AI, Human in the Loop, and Flow
- The **HTTP Request node** is your universal connector — if n8n doesn't have a specific node for something, HTTP Request can still connect to it
