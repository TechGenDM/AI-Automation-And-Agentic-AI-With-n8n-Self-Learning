## Part 1: Setting Up n8n

### 3 ways to run n8n

| Way | Think of it like... | Best for |
|---|---|---|
| **n8n Cloud** (official website) | Excel Online — no install, just sign up | Beginners (this is what we use in this course) |
| **Local (via Docker)** | Excel installed on your own PC | People who want it free forever, on their own laptop |
| **Your own server** (AWS/GCP/Azure) | Installing Excel on a company server | Production use, businesses |

**For learning → use Cloud.** You get:
- **14-day free trial**, **1,000 free executions** (an "execution" = one time your workflow runs)
- No credit card needed
- After the trial ends, you can just make a new account — no limit on how many trial accounts you can make

### Steps to sign up

1. Go to `n8n.io` → click **"Get started for free"**
2. Enter email + password
3. Pick a **unique account name** — this becomes your URL, like `yourname.app.n8n.cloud`
4. Answer a few onboarding questions (company size, role, etc.) — doesn't matter much, pick whatever's closest
5. Skip "invite team members" (not needed when working alone)
6. You land on your workflows dashboard 🎉

**Extra option:** There's also a **free self-hosted "Community Edition"** on GitHub — no cost, no execution limits, but needs some setup effort (this is the "Local via Docker" option above).

## Part 2: The Interface (Dashboard)

Think of the dashboard like Excel Online's home screen — before you even open a spreadsheet, there's a home page with your files, settings, and account.

| Section | What it's for |
|---|---|
| **Your name (top-left) → Settings** | Your profile, dark mode, 2FA (turn this on for security) |
| **Overview page** | Shows executions from last 7 days, failure rate, time saved |
| **Personal / Projects** | Like folders — organize your workflows (e.g., a "Dev" folder vs a "Prod" folder) |
| **Variables** | Store a value once (like an API key) and reuse it everywhere — so you don't paste the same key into 10 different workflows |
| **Templates** | 5,000+ pre-built workflows shared by other people |
| **Manage → Settings** | Time zone, version upgrades, community nodes (extra building blocks beyond the default ones) |

### ⚠️ Important advice
Don't just grab a template and run it thinking you've "learned automation." People who skip straight to templates never understand the actual building blocks — and later get stuck, thinking "no-code doesn't work," when really they just skipped the fundamentals.

**Build from scratch first. Use templates later, once you understand the basics.**

## Part 3: The Canvas (where you actually build)

Think of the Canvas as your **drawing board** — an empty space where you drag and connect building blocks called **"nodes."**

### Key controls (worth memorizing)

| Shortcut | Action |
|---|---|
| Mouse wheel / pinch | Zoom in/out |
| `0` | Reset zoom to default |
| `1` | Center your view on all nodes |
| `Ctrl/Cmd + S` | **Save** — nothing autosaves! If you close without saving, your work is gone |
| `Tab` | Open the node search panel |

### Other important things

- **Sticky Notes**: You can add text notes directly on the canvas (written in Markdown) — very useful for documenting your own workflows, just like comments in code.
- **Download as JSON**: Every workflow can be exported as a `.json` file. This is exactly how workflow files get shared between people — including the ones used in this course.
- **AI Assistant**: A built-in chat helper (still in beta) that can answer "how do I...?" questions using n8n's own documentation.

---

## Key Takeaway

- Use **n8n Cloud** to start learning (free trial, no card needed)
- The **dashboard** is your home base — Projects, Variables, and Templates live here
- **Never skip straight to templates** — build from scratch first so you actually understand the building blocks
- The **Canvas** is where the real work happens — remember `Ctrl+S` to save, and use Sticky Notes to document your workflows
