## Research Agent

One of the main additions in this module is the research agent.

The purpose of this agent is to research a topic on the internet or from available knowledge tools.

The workflow is designed so that the agent can:

- receive a user query
- decide which tool is appropriate
- gather information from a knowledge source
- return a helpful response

### Why this matters

A research agent is useful because it can go beyond simple conversation and provide context-based answers.

Examples include:

- researching a company or topic
- collecting recent information
- finding facts or definitions
- summarizing search results

## Tools Used in the Research Agent

The instructor demonstrates a research workflow using tools such as:

- Wikipedia
- a web search-style tool
- an HTTP request tool

The main point is not just the specific tools, but the pattern:

> The agent should choose the correct tool based on the user’s request.

So if the user asks for factual background, the agent may use a knowledge tool. If the user asks for recent updates, it may use a search-based tool.

This is an important concept in agent design: tools are the way the AI extends its capability.

## Understanding the Agent Flow

The lesson also explains how the internal flow of an agent works.

The request goes through several layers:

1. User asks a question.
2. The agent receives the message.
3. The agent checks its instructions and available tools.
4. It chooses the most suitable tool.
5. It collects the data.
6. It produces a final answer.

This is a very important mental model for understanding agents in n8n.

It helps you see that an agent is not just a chatbot. It is a system that combines:

- instructions
- tools
- memory
- reasoning

## Memory in the Agent

The module also highlights the role of memory.

Memory helps the assistant retain context from previous messages and use that context in future steps.

This is especially useful when:

- the user asks follow-up questions
- the agent needs prior context
- the conversation becomes multi-turn

So memory is what allows the assistant to feel more intelligent and conversational.

## Mocking API for Your Application

A major part of this module is about mocking APIs.

The instructor explains that in real development, you may not always have a live API already ready. Sometimes you want to test your workflow quickly before building a full backend.

That is where mock APIs become useful.

### What is a mock API?

A mock API is a fake or temporary API endpoint that returns predefined data.

It helps you:

- test your workflow
- simulate an application response
- build and debug without needing a full production backend

The instructor uses a service like Beeceptor as an example.

With it, you can create a mock endpoint and return simple JSON responses.

This is very practical because it lets you test how an AI agent would interact with an external service even before that service exists fully.

---

## Why Mock APIs Are Useful

Mock APIs are useful in several scenarios:

- you are building a prototype
- you want to test API integration quickly
- you do not want to wait for a real backend
- you want to simulate a third-party service
- you want to debug the workflow logic safely

This is a very beginner-friendly but highly practical approach to development.

## HTTP Node for Third-Party Services

The lesson also introduces the HTTP Request node as an important way to connect n8n to external services.

The key idea is:

> If a service is not available as a built-in n8n app, you can still connect to it using its API endpoint.

That is how the agent can interact with services that are not natively integrated into n8n.

The workflow uses the HTTP node to send requests to the mock endpoint and receive data.

This is a very important skill because many real-world tools are exposed through APIs.
