## Part 1: Automation vs AI Agents (sharper version)

**The core distinction, in one line:**
> Automation = deterministic (rules never change) | AI Agent = non-deterministic (adapts based on understanding)

### The coffee machine example
- **Automation**: Coffee brews at 7am. Doesn't matter if you're home, asleep, or on vacation — it still runs. Rigid.
- **Agent**: Understands you're not up yet (checks the environment) and makes coffee at 10am instead, when you actually wake up. Adaptive.

### The universal flow (applies to both)
```
Input/Trigger → Action(s) → Output
```
The **input and output can look identical** in both cases (e.g., "someone messages you" → "you get a reply"). What differs is **what happens in the middle**:
- Automation's middle = fixed, predefined steps
- Agent's middle = a "brain" (LLM) + memory + tools, making its own decisions

### Important teaching rule
> "Don't unnecessarily include AI/agents if it's not required." A signup → welcome email doesn't need AI. Save agents for where adaptability actually matters.

**Course strategy**: Master plain automation completely first → then move to agents.

## Part 2: Building Your First Real Workflow — Lead Capture System

### The goal
```
Someone fills a form → their info saved to Google Sheets → you get an email notification
```

### Step 1: Organize first (good habit)
Before building, create a **Project** (e.g. "n8n Foundations") and a **Folder** inside it — keeps workflows tidy as you build more of them. Always add a **Sticky Note** at the start of a workflow explaining what it does — like commenting your code.

### Step 2: Build the Trigger — a Form
Use the **n8n Form Trigger** node instead of a Manual Trigger:
- Set a form title + description
- Add form fields — e.g., **Name** (text) and **Service** (dropdown)
- Mark required fields
- Test with **"Execute step"** — this gives you a live test URL you can actually fill out like a real user would

This is functionally just like a Google Form — just a way to collect structured input that becomes your workflow's starting data.

### Step 3: Connect to Google Sheets (save the lead)
This introduces a key recurring concept: **Credentials**.

- n8n can't access your Google Sheet without your permission — you authorize it (OAuth2 sign-in with Google), just like any app asking "Allow access to your Google Drive?"
- Once connected, you configure:
  - **Resource** = Sheet (vs whole Document)
  - **Operation** = "Append Row" (add a new row each time)
  - Which **Document** (spreadsheet) and **Sheet** (tab) to write to

### Step 4: The Golden Trick — Drag & Drop Data Mapping
This is the most important practical skill in this module:

- Each Google Sheet column (Name, Service, Time) needs a value
- Instead of typing static text, **drag the field from the previous node** (the Form) directly into the column box
- The moment you drag something in, the field switches from **"Fixed" mode to "Expression" mode** — meaning it's now dynamic, pulling in whatever the user actually typed into the form

This is the **Data Flow** concept in action — data (JSON) flowing from one node into the next.

### Step 5: Send the Email (Gmail node)
- Connect Gmail via credentials (same OAuth pattern as Sheets)
- Set **To**, **Subject**, and **Message**
- Same drag-and-drop trick: pull in `Name`, `Service`, etc. from earlier nodes so the email is personalized per lead, not generic

### Step 6: Test end-to-end
Execute the workflow → check the Google Sheet got a new row → check the inbox got the email. ✅ Working automation!

## Key Concepts to Remember

| Concept | What it means |
|---|---|
| **Credentials** | Every app connection (Gmail, Sheets, etc.) needs authorization — one-time OAuth setup, reused across workflows |
| **Fixed vs Expression mode** | Fixed = hardcoded value. Expression = dynamic, pulled from a previous node's data |
| **Drag & drop mapping** | The easiest way to pass data between nodes — no manual syntax typing needed |
| **Sticky Notes** | Document your workflow as you build — future-you will thank you |

---

## Key Takeaway

- Automation = fixed rules, Agent = adaptive brain — don't force AI where it's not needed
- Your first real workflow: **Form Trigger → Google Sheets (Append Row) → Gmail (Send)**
- **Credentials** connect n8n to real apps (one-time setup via OAuth)
- **Drag and drop** is how you pass data between nodes — it automatically creates a dynamic "Expression"
